Instructor Guide

Inspiration

  • Executed the workshop at the Home Depot multiple times (15 - 20) and then eventually taken over by the Home Depot.
  • Teach them to talk to each other. This is not just about PCF.
  • This workshop is for everyone (not just the leads).
  • Teach people how to learn and talk to each other.
  • Respect and empathy are critical. They may have questions like: What’s my role going to be? Will I have a job?
  • Create a safe place for people
  • 15 people average size - smaller don’t talk to each other, larger you will be running around all over the place
  • Get a single team that is focused on a task

Methodology

Modules are designed to have a give and take with the students. Presentation → Lab → Recap. Everyone should get a break this way. The presentations are executed with PowerPoint and should be conversational, then students complete the labs and are followed by a recap of the module with a discussion. This is supplemented with instructor notes that have specific questions to discuss as a group. Instructor notes are hidden, but follow a specific convention to access them. For each lab url append /notes.

For example:

/pivotal-cloud-foundry-developer/logging-scale-ha → /pivotal-cloud-foundry-developer/logging-scale-ha/notes

Please feel the freedom to execute as you see fit for a given class. For instance, allowing students to get ahead in the labs.

Setup Script

The following script is can be used to setup/create PCF users and set permissions. It has to change with the email format. I recommend testing one email first, to make sure it creates users/orgs as expected. Make sure you are logged in as admin to PCF prior to running.

#!/usr/bin/env bash
#
# This script requires bash 4! Upgrade!
#
# This script takes in a list of emails (newline seperated) and builds out cf orgs and users.
# It is assumed that the email takes the form of first_last or first.last. We will create an
# org named fl-org where f is the first initial and l is the last initial. If we encounter
# 1 duplcate we will append 1 to the org name. If we encounter two duplicates your in trouble
# this script creates org_summary.txt which is helpful for orienting students about where they
# should go.
#



declare -A orgs

echo `date` > org_summary.txt
for email in `cat class-participant-emails.txt` ; do
	org=`echo $email | tr '_.' "\n" | egrep -v com | colrm 2 | tr -d "\n" && echo -org`

	if [[ ${orgs[$org]} -eq 1 ]]; then
		org=$org"1"
	fi
	orgs[$org]=1

	echo "INFO: Setting up $org: for $email" | tee -a org_summary.txt

	cf create-org $org
	cf create-space development -o $org

	cf create-user $email password
	cf set-space-role $email $org development SpaceDeveloper

done
